# 恋愛シミュレーション・陰陽寮占いモード 引継ぎ計画書

最終更新: 2026-06-30  
基準コミット: `1f3a82a`  
対象: `寝殿造り3D探訪_統合版.html`

この文書は、今後「平安貴族の恋愛シミュレーション」と「陰陽寮の占いモード」を実装するための詳細な引継ぎ計画書です。Codex、Claude、別チャット、別AIにそのまま渡しても、意図・実装順・既存コードとの接続点がわかるように書いています。

## 0. 最重要ルール

- 正本は GitHub `origin/main`。作業前と push 前に必ず `git fetch origin` する。
- 本体は単一HTML `寝殿造り3D探訪_統合版.html`。分割ビルドや外部ライブラリ追加は避ける。
- Three.js は r128。`outputEncoding` / `sRGBEncoding` を維持し、`colorSpace` API に置換しない。
- 通常散策時の毎フレーム負荷を増やさない。恋愛・占い・ミニゲーム中だけ必要な判定を動かす。
- 既存モードを壊さない。特に `APP.mode`、`enterMode()`、`updateControlUI()`、`setTime()`、`applySeason()`、`AmbientAudio`、`SFX`、`result` ダイアログを触る時は最小差分にする。
- Claude/Codex 同時編集時は巨大HTMLの同じ領域を同時に触らない。push 前に `origin/main` との差分を必ず確認する。

## 1. 目標

新しい大きな柱として、平安時代の恋愛常識を「遊びながら理解する」モードを作る。

学ばせたいこと:

- 平安貴族の恋愛は、直接告白ではなく、噂、垣間見、和歌、文、返歌、女房の仲介、牛車移動、季節感、方角・吉凶、評判によって進む。
- 和歌は単なる飾りではなく、相手の教養・季節感・誠実さを測るコミュニケーションだった。
- 女性に直接会うことは難しく、御簾・几帳・女房・侍従・時間帯が関係を左右した。
- 陰陽道、物忌み、吉方、夢見、怪異は、日常判断や恋愛行動にも影響した。

追加したいモード:

1. `恋文ショート体験`
   - 10〜15分で恋愛の流れを体験する短編。
   - 失敗しても学べる、テンポ重視。

2. `恋愛シミュレーション`
   - 30日、90日、または1年を進める長期モード。
   - パラメータを育て、意中の女性の好感度を上げ、恋を成就させる。

3. `陰陽寮の占い`
   - 散策中にランダム発生。
   - 恋愛シミュレーション内では日付ごとに吉凶として組み込む。
   - 方違え、物忌み、夢見、吉方、凶方、怪異の兆しを扱う。

## 2. 既存コードで使うべき接続点

検索アンカー:

- `const APP={...}`  
  新モード用の状態を追加する。例: `renai:null`, `onmyo:null`。

- `enterMode(m)`  
  `renaiShort`, `renaiSim`, `onmyo` を追加する。トップバー・HUD表示・音の切替をここで制御する。

- `#title` / `mainModes` / `taikenSubPanel` / `miniGameSubPanel`  
  メインメニューに `恋愛モード` か `物語モード` の区分を追加する。

- `#result` / `resTitle` / `resRank` / `resDetail` / `resKaimamiActions`  
  ショート体験と長期モードの結果表示に流用する。現在は `resTop` があるので表紙へ戻る導線は使える。

- `setTime(t)` / `applySeason(season)`  
  恋愛モードの日付進行、季節イベント、時間帯演出に使う。

- `WAKA_DATA`  
  既存100首を教材・例歌・季節判定に使う。プレイヤー生成和歌は別データで管理する。

- `kaimami` 系  
  文の配達、垣間見、忍び訪問イベントの土台にできる。ただし既存の垣間見ミッション本体を壊さず、恋愛用の簡易イベントから始める。

- `SFX` / `AmbientAudio`  
  返歌受領、牛車移動、香合わせ、占い札、怪異兆候などのSEに使う。外部音源追加は最小限。

- `PREFS`  
  長期モードの途中セーブは新キーに分ける。例: `shinden3d-renai-save-v1`。

## 3. 推奨する新規状態設計

最初はクラス化しすぎず、既存コードの書き方に合わせてプレーンオブジェクトで始める。

```js
APP.renai = {
  mode: "short",          // "short" | "sim"
  day: 1,
  maxDay: 30,
  phase: "intro",         // intro, rumor, kaimami, poem, reply, visit, result...
  time: "day",
  season: "spring",
  targetId: "aoi",
  stats: {
    miyabi: 10,           // 雅
    poetry: 10,           // 文才
    reputation: 10,       // 評判
    modesty: 10,          // 慎み
    incense: 10,          // 香道
    fashion: 10,          // 装束
    spirit: 10,           // 霊力
    wealth: 10            // 財力
  },
  affection: {
    aoi: 0,
    yuki: 0,
    tachibana: 0
  },
  flags: {},
  log: []
};

APP.onmyo = {
  today: null,
  lastRollDay: 0,
  omenQueue: []
};
```

注意:

- `APP.time` / `APP.season` と恋愛モード内部の日付・季節が二重管理にならないよう、モード開始時に現在値を保存し、終了時に戻す。
- 再現モードと同じ考え方で `prevSeason`, `prevTime`, `prevLabelMode`, `prevAutoPaused` を保存する。
- 長期モードの保存は最初から localStorage に入れる。ただし巨大なログを保存しすぎない。

## 4. モード構成

### 4.1 恋文ショート体験

目的:

- 平安恋愛の流れを短時間で体験する。
- 学習導線として、説明ではなく「行動→反応→解説」で理解させる。

推奨フロー:

1. 噂を聞く
   - 女房や侍従から「ある姫君は春の歌を好む」と聞く。
   - 学習: 当時の恋愛は噂と仲介者が重要。

2. 牛車で移動
   - 邸の近くまで移動する短い演出。
   - 選択肢: 時間帯、方角、同行者。
   - 占いが絡む場合「今日は東が吉」などを提示。

3. 垣間見
   - 既存の垣間見ミッションより簡単な、視線合わせ・距離だけの短い判定。
   - 成功すると相手の性格ヒントを得る。

4. 和歌を作る
   - 完全自由入力ではなく、語句カード選択式。
   - 例: 季語、景物、恋情、結句を選ぶ。
   - 季節・相手の好み・慎みで評価。

5. 文を送る
   - 女房経由で送る。直接届けると評価が下がる場合がある。
   - 簡易ミッション: 誰に託すかを選ぶ。

6. 返歌を受け取る
   - 相手の好感度によって返歌の温度が変わる。
   - 返歌後に短い解説を出す。

7. 香・装束を整える
   - 香合わせか装束コーデのどちらかを1回だけ体験。

8. 結果
   - `好印象`, `まだ脈あり`, `無粋`, `噂になりすぎ` など。
   - `resDetail` に学習ポイントを出す。

MVPでは 1本道でよい。分岐は「良い/普通/失敗」の3段階で十分。

### 4.2 恋愛シミュレーション 長期モード

目的:

- 30日または1年を進め、意中の女性の好感度を上げる。
- 行事・吉凶・噂・文・贈り物・怪異・牛車移動を組み合わせる。

基本ループ:

1. 朝: 陰陽寮の占い、噂、予定確認
2. 昼: 教養上げ、香合わせ、装束準備、贈り物準備、文作成
3. 夕: 牛車移動、文の配達、行事参加
4. 夜: 忍び訪問、返歌確認、怪異調査、物忌み
5. 日付進行: パラメータ・評判・好感度を更新

勝利条件:

- 指定日までに対象女性の好感度が一定以上。
- 評判が低すぎない。
- 慎みが低すぎない。
- 最終訪問イベントで成功する。

敗北/未達:

- 噂が悪化しすぎる。
- 返歌が途絶える。
- 物忌みや凶方を無視し続けてイベント失敗。
- 怪異イベントを放置して邸に近づけなくなる。

推奨パラメータ:

- `雅`: 季節感・所作・行事選択に影響。
- `文才`: 和歌・返歌の評価に影響。
- `評判`: 噂、他貴族、女房の態度に影響。
- `慎み`: 強引な訪問や連続文で下がる。高いと信頼されやすい。
- `香道`: 香合わせ、贈り物、訪問前準備に影響。
- `装束`: ファッションコーデ評価。季節・時間帯・場に合うほど上がる。
- `霊力`: 怪異調査、陰陽イベントに影響。
- `財力`: 牛車、贈り物、行事参加に影響。

対象女性の個性:

```js
const RENAI_TARGETS = {
  aoi: {
    name: "葵の君",
    taste: { poetry: 1.3, modesty: 1.2, fashion: .8 },
    favoriteSeason: "spring",
    dislikes: ["too_direct", "night_noise"],
    intro: "御簾の内で物語を好む、慎み深い姫君。"
  },
  yuki: {
    name: "雪の君",
    taste: { poetry: 1.1, incense: 1.3, spirit: 1.1 },
    favoriteSeason: "winter",
    dislikes: ["bad_omen", "cheap_gift"],
    intro: "香と雪景色を愛し、夢見を重んじる姫君。"
  }
};
```

最初は2人でよい。増やす時は性格差がゲーム差になるようにする。

## 5. 和歌生成・返歌システム

### 5.1 基本方針

- 完全自由入力は避ける。評価が難しく、スマホで入力しづらい。
- 語句カード選択式にする。
- 既存 `WAKA_DATA` は「お手本」「季節例」「図鑑連携」に使う。
- プレイヤーの和歌は短い疑似和歌でよい。ただし縦書き表示と雅な演出で満足感を出す。

### 5.2 カード分類

```js
const RENAI_WAKA_CARDS = {
  season: [
    {id:"sakura", text:"桜", season:"spring", miyabi:2},
    {id:"hotaru", text:"蛍", season:"summer", miyabi:2},
    {id:"momiji", text:"紅葉", season:"autumn", miyabi:2},
    {id:"snow", text:"雪", season:"winter", miyabi:2}
  ],
  feeling: [
    {id:"shinobu", text:"忍ぶ恋", modesty:2},
    {id:"matsu", text:"待つ夜", poetry:1},
    {id:"aware", text:"あはれ", miyabi:1}
  ],
  image: [
    {id:"moon", text:"月影", season:null, miyabi:1},
    {id:"dew", text:"露", season:"autumn", poetry:1}
  ],
  ending: [
    {id:"to_you", text:"君に知られむ", direct:1},
    {id:"hidden", text:"袖に隠しつ", modesty:1}
  ]
};
```

### 5.3 評価ロジック

評価軸:

- 季節一致: 現在季節と季語が合う。
- 相手好み: 対象女性の好む軸に合う。
- 慎み: 初期段階で露骨すぎない。
- 文才: プレイヤーステータス補正。
- 吉凶: 今日の占いが文や訪問に吉なら成功率上昇。
- タイミング: 返歌後すぐ連投しない。夜に送るか昼に送るか。

擬似式:

```js
score =
  seasonBonus +
  targetTasteBonus +
  stats.poetry * 0.25 +
  stats.miyabi * 0.18 +
  stats.modesty * 0.12 +
  onmyoBonus -
  tooDirectPenalty -
  badTimingPenalty;
```

返歌:

- score 高: 相手も季節に応じた美しい返歌。
- score 中: 丁寧だが距離のある返歌。
- score 低: 返歌が遅い、女房からやんわり断られる。
- score 最低: 評判低下、次回訪問難度上昇。

## 6. 年中行事カレンダー

長期モードでは日付を進める。最初は「抽象日付」でよい。

推奨:

- 1年モードは大きすぎるため、MVPは30日。
- `day` から季節を算出し、特定日に行事イベントを出す。
- 将来、1年モードへ拡張する。

イベント例:

| 日 | 季節 | 行事 | 効果 |
|---|---|---|---|
| 3 | 春 | 花見 | 雅・文才が上がる。春好みの姫に有効 |
| 7 | 春 | 曲水の宴 | 和歌勝負イベント |
| 12 | 夏 | 納涼 | 牛車移動・夜訪問に好機 |
| 15 | 夏 | 七夕 | 恋文成功率上昇 |
| 20 | 秋 | 虫聞き | 「能は歌詠み」再現モードと連携可能 |
| 24 | 秋 | 重陽 | 菊・長寿・香イベント |
| 27 | 冬 | 雪見 | 冬好みの姫に有効 |
| 30 | 冬 | 最終訪問 | エンディング判定 |

散策側の季節表現と同期するため、日付が進む時に `applySeason()` を呼ぶ。

## 7. 文の配達・秘匿ミッション

既存の垣間見ミッションを参考に、軽量なミッションとして作る。

MVP:

- 画面上に「女房」「侍従」「門番」から仲介者を選ぶ。
- 仲介者ごとに成功率と評判リスクが違う。
- ステルス移動は後回し。

発展:

- 邸の外から文を届ける短い移動ミッション。
- 見つかると評判低下。
- 信頼済みの女房を使うと成功率上昇。

学習ポイント:

- 高貴な女性への接触は女房の仲介が重要。
- 直接的すぎる行動は無粋、あるいは危険。

## 8. 牛車移動

MVP:

- 行き先・時間帯・方角を選ぶ。
- 短いフェード/カメラ移動/牛車SEで演出。
- 吉方なら成功率上昇、凶方ならイベント失敗率上昇。

選択肢:

- 行き先: 姫君の邸の近く、宮中、寺社、陰陽寮、賀茂祭
- 時間帯: 昼、夕、夜
- 方角: 東西南北

将来:

- 3D上に牛車モデルを動かす。
- 道中で噂、怪異、行事イベントが発生する。

## 9. 香合わせミニゲーム

MVP:

- 3つの香から、相手好みや季節に合うものを選ぶ。
- 正解なら `香道` と好感度上昇。
- 間違っても学習解説を出す。

例:

- 春: 梅、桜、若草
- 夏: 蓮、涼風、蛍
- 秋: 菊、紅葉、萩
- 冬: 梅、雪、沈香

UI:

- 設定画面ではなく恋愛モード専用の小パネル。
- カード3枚。香名、短い説明、選択後に香炉エフェクト。

## 10. 装束コーデミニゲーム

MVP:

- 季節、時間帯、訪問先に合う装束を選ぶ。
- 色合わせと場違い判定をする。

評価:

- 季節色: 春=薄紅/萌黄、夏=白/浅葱、秋=紅葉/朽葉、冬=白/濃色
- 場: 宮中、私邸、夜訪問、行事
- 相手好み: 華やか/控えめ/知的

学習:

- 平安の装束は身分・季節・場と密接に関係する。

実装は当面2Dカードでよい。3Dモデル着せ替えは後回し。

## 11. 怪異調査ミッション

恋愛と物の怪退治を接続する要素。

使い方:

- 姫君の邸に怪異の噂が立つ。
- 放置すると夜訪問不可、好感度停滞、評判悪化。
- 調査すると霊力・評判が上がる。
- 既存 `taiji` へ直接飛ばすのではなく、最初は簡易調査にする。

MVP:

1. 噂を聞く
2. 陰陽寮で占う
3. 痕跡を選ぶ
4. 札を置く
5. 成功/失敗

発展:

- 物の怪退治モードのボス戦へ分岐。
- 安倍晴明NPCと連携。

## 12. 陰陽寮の占いモード

### 12.1 散策中ランダム発生

散策中に低確率で占いイベントを出す。

発生条件:

- `APP.mode === "walk"`
- 前回発生から一定時間経過
- 夜・朝・特定季節で確率変動
- ユーザー操作を邪魔しない小さなトースト/チップから開始

イベント例:

- 「陰陽師が今日の吉方を告げています」
- 「夢見が悪い。北へ向かう前に祓え」
- 「物忌みの日。無理な訪問は慎むべし」

報酬:

- 図鑑項目「陰陽寮」「方違え」「物忌み」解放
- 恋愛モードで使える小さなバフ

### 12.2 恋愛シミュレーション内

毎朝、今日の占いを生成する。

```js
const ONMYO_OMENS = [
  {id:"east_lucky", label:"東吉", goodDir:"east", effect:{letter:1.1, travel:1.15}},
  {id:"monoimi", label:"物忌み", forbid:"visit", effect:{visit:.7, study:1.2}},
  {id:"bad_dream", label:"悪夢", effect:{spiritMission:1.2, visit:.85}},
  {id:"moon_good", label:"月徳", time:"night", effect:{poetry:1.15}}
];
```

占いは強制ではなく、無視もできる。ただし無視するとリスクが増える。

### 12.3 UI

- 散策中: 小さな通知 + 詳細ボタン。
- 恋愛モード: 朝の画面に「今日の吉凶」カード。
- 設定パネルには入れない。ゲーム内イベントとして扱う。

## 17. 画面とスマホ対応

スマホ縦画面を基準に設計する。

- 選択肢は最低44px以上。
- 和歌カードは2列まで。長文はカード下に説明を出す。
- HUDは上部固定しすぎない。ボス戦UIのように専用表示にするならトップバーは隠す。
- `result` ダイアログを流用する時は、ボタンが3つ横に並ばない場合は折り返す。
- 長期モードでは情報量が多いので、`今日`, `文`, `装束`, `占い`, `記録` のタブUIがよい。

## 19. リスクと回避策

リスク:

- 巨大HTMLにさらに機能を足しすぎて見通しが悪くなる。
- 長期モードを最初から作り込みすぎて完成しない。
- 説明文が多くなり、ゲームではなく教材画面になる。
- 和歌評価が不透明で理不尽に感じる。
- スマホUIが情報過多になる。

回避策:

- まずショート体験を完成させる。
- 長期モードは30日MVPに限定する。
- 1画面1判断にする。
- 評価後に「何が良かった/悪かった」を短く表示する。
- パラメータは最初8つあるが、画面表示は主要3つだけに絞る。

## 20. 実装時の推奨ファイル内アンカー

追加位置の目安:

1. CSS:
   - `/* ===== 上部バー ===== */` の後ではなく、既存HUD群の近くに `/* ===== 恋愛モード ===== */` を新設。

2. HTML:
   - メインメニュー: `mainModes` 周辺。
   - HUD: `questHud`, `taijiHud`, `saigenHud` 付近。
   - 結果画面は既存 `#result` を流用。

3. JS:
   - 状態: `const APP={...}`。
   - データ: `WAKA_DATA` の近くではなく、恋愛モード用に `/* 恋愛シミュレーション */` セクションを作る。
   - 起動: `enterMode()` とタイトルボタン群。
   - 更新: `animate()` に毎フレーム処理を入れる場合は、`if(APP.mode==="renaiSim")` で限定する。
