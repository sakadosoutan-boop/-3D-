# 非恋愛モード品質向上計画

最終目標: 恋愛モードに触れず、既存の各モードを「単体で遊べる商品」水準へ段階的に磨く。

## 作業対象外

- `renaiShort`
- `renaiSim`
- `renaiHud`
- 恋愛モード用BGM / ターゲットピッカー / 恋文演出

同時作業中のため、恋愛モード周辺のHTML/CSS/JSは原則編集しない。

## 優先度A: 全モード共通の遊びやすさ

1. 開始直後に「目的・上達目標・操作」を短く見せる任務カードを出す。
2. 結果画面に「次に狙う目標」を出し、再挑戦理由を明確にする。
3. モード選択カードに、所要時間・難度・学習効果を明示する。
4. スマホ縦画面では、HUDと操作ボタンが重ならないか毎回スクリーンショットで確認する。

今回実装済み:

- 散策・クイズ・一日体験・垣間見に任務カードを追加。
- 恋愛モードは `body.renai-mode #modeBrief{display:none}` とし、明示的に除外。

## 優先度B: 散策モード

- 図鑑未発見の対象を季節・場所・分類で自然に誘導する「発見ログ」を追加。
- 季節ごとの動物・虫・鳥に、視線を引く小さな動きと音を追加。
- 建築部位を調べた後、関連部位へつながる「次に見ると面白いもの」を提示。
- 写真モードの構図補助、被写界深度風の軽い演出、保存用UIを強化。

## 優先度B: クイズモード

- 正解時に対象へ短いカメラ寄せを入れ、場所と名称を記憶に残す。
- 誤答時に「似ている部位との差」を1行だけ表示する。
- 結果画面に苦手分類の分析を追加する。
- タイムアタックはゴースト目標タイム、連続正解ボーナス、称号演出を追加する。

## 優先度B: 一日体験モード

- 男性 / 女性でカメラ演出と目的地演出を差別化する。
- 各行動の文化的意味を、説明文ではなく場面中の所作と短い台詞で見せる。
- 達成後に「今日覚えた平安常識」を3枚カードで振り返る。
- 牛車・御簾・几帳・楽器など、重要所作のミニインタラクションを増やす。

## 優先度A: 垣間見ミッション

- 視線の予兆を、赤い画面端だけでなく従者の顔向き・足音・扇の動きで示す。
- 物陰に入った時の安心感を、音量・画面色・危険メーターの戻りで演出する。
- 三つの観察地点それぞれに短いカットを追加し、文学的な意味を違う角度で見せる。
- 見つかった時は即リセットだけでなく、原因がわかるリプレイ風の一言を出す。

## 優先度A: 物の怪退治

- 各ボスの攻撃に「予兆 → 回避余地 → 着弾演出 → 余韻」を徹底する。
- 雪女、九尾狐、河童、提灯お化け、人魂 / 鬼の攻撃テンポを差別化する。
- BGM、SE、ヒットストップ、画面揺れを軽量に統一管理する。
- 戦闘UIは市販アクションゲーム水準で、HP、スタミナ、武器、回避、ガード、ロック対象が一目でわかるようにする。
- スマホでは親指の自然位置に合わせて、移動・攻撃・回避・ガードを同時操作できる配置を維持する。

## 優先度B: 歌合モード

- 判者の好みを試合前に少しだけ匂わせ、選歌の読み合いを作る。
- 札選択時に、題・季節・技巧との相性を視覚的に出す。
- 勝敗後に判詞をより短く、強く、古文知識として残る文章にする。
- 雅楽BGMの入りと止めを場面転換に合わせて調整する。

## 優先度B: 再現モード

- 本文の段ごとに、役者の立ち位置、視線、手の動き、建具の状態を表で管理する。
- 漫画的なカット割りと3D空間の見やすさを両立するため、手すりや御簾の透明度を場面別に調整する。
- 和歌表示は縦書き2行を基本に、本文量が多い場面は上揃えで収める。
- 虫や人物は「見せたいものだけ」を残し、不要な小物を減らして軽量化する。

## 検証方針

- HTML構文エラー確認。
- Playwrightで最低限、タイトル、散策、クイズ、一日体験、垣間見の開始導線を確認。
- 変更前に `git fetch origin` し、Claude / Codexの最新変更を取り込む。
- 恋愛モード関連の差分が混ざっていないか `git diff` で確認する。
