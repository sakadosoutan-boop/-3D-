# ストーリーモード 舞台仕様書（STAGE KIT）

最終更新: 2026-07-02
対象: `寝殿造り3D探訪_統合版.html` の再現エンジン（SAIGEN_*）

大規模ストーリーモードの脚本は、**再現モードのシーン形式（SAIGEN_SCENES）をそのまま使って書けます**。
この文書は「脚本に何が書けるか（＝舞台側が何を用意済みか）」の全仕様と、既存シナリオの校閲記録です。
Codex・Claude・別チャットにそのまま渡せます。

## 0. 最重要ルール（既存の開発規約と同じ）

- 正本は GitHub `origin/main`。作業前と push 前に必ず `git fetch origin`。
- 単一HTML。Three.js r128（`outputEncoding`/`sRGBEncoding` 維持。`colorSpace` API 禁止）。
- 恋愛モード（renaiShort / renaiSim / renaiHud）はCodex担当領域。原則触れない。
- 通常散策の毎フレーム負荷を増やさない。ストーリー演出はモード中だけ動かす。

## 1. シーンの書き方（1作品＝1オブジェクト）

`SAIGEN_SCENES` 配列に追加するだけで目次（作品ピッカー）に自動で並ぶ。

```js
{
  id:"story_XX",            // 一意ID
  work:"作品名",            // 例:"古今著聞集"
  title:"章題・場面名",
  author:"作者・出典",
  unit:"段",                // カウンタ表示の単位（"段"/"場"/"幕"など）
  world:true,               // true=実際の寝殿(母屋・廂)を舞台に使う / 省略=南庭の仮設舞台
  buildProps:"chomon",      // SAIGEN_PROP_BUILDERS のキー（場面専用の道具一式）
  cricket:true,             // 専用環境音（キリギリス）。他の環境音は自動停止
  cast:[ ... ],             // 配役（下記2章）
  beats:[ ... ]             // ビート＝紙芝居の1コマ（下記3章）
}
```

## 2. 配役（cast と SAIGEN_CHARS）

### 2.1 登場人物プリセット（SAIGEN_CHARS 定義済み）

| key | 名前 | role | 用途 |
|---|---|---|---|
| seishonagon | 清少納言 | nyobo | 枕草子 |
| sadaijin / samurai / nyoboL / nyoboR | 左大臣・侍・女房×2 | — | 能は歌詠み |
| tomonori / umakata | 紀友則・右方の人 | — | 寛平歌合 |
| okina / ona / kaguya | 翁・媼・かぐや姫 | — | **竹取用ひな形（未使用）** |
| otoko / onna | 男・女 | — | **伊勢物語用ひな形（未使用）** |

新キャラは1行追加で作れる:
`key:{name:"表示名", role:"kikoshi|himegimi|nyobo", palette:[色…], scale:1.0, prop:"scroll|fan|shaku"|null, standing:true?}`
`standing:true` は立位（侍など）。座位モデルを自動で立ち姿に補正する。

### 2.2 cast の配置

- `world:true` のシーン: `{key, group:"moya", pos:[x,y,z], yaw, scale, label:"頭上ラベル"}`（実座標。母屋の床はy≈1.30、廂はy≈1.20）
- 仮設舞台のシーン: `{key, dx, dz, yaw, scale}`（舞台中心からの相対）
- `group` を分けると場面グループ切替で自動的に出し分けられる（例: "moya"と"uta"）。

## 3. ビート（beats[]）に書けるフィールド全一覧

### 3.1 テキスト系
| フィールド | 内容 |
|---|---|
| `honbun` | 古文本文（縦書きパネルに表示） |
| `yaku` | 現代語訳 |
| `note` | 語釈・文法・文化注（「〔注〕」付きで表示） |
| `speaker` | 台詞の主のkey（`null`=地の文）。**慣例: 引用台詞の話者を入れる** |
| `waka:{lines:[上句,下句], author}` | 和歌の中央大表示（縦書きカットイン）。chapterと併用不可 |
| `bubbles:[{key,text,cls}]` | 吹き出し。`\n`で2行。clsは下記3.4 |

### 3.2 舞台系
| フィールド | 内容 |
|---|---|
| `season` / `time` | 季節・時間帯を切替（世界の空・光・草木が変わる） |
| `group` | 場面グループ（castのgroupと対応。変えると場転換） |
| `hide:["key"]` | この場に「まだ登場しない」人物を隠す |
| `focus:"key"` | 話者を強調（スポットライト・微拡大） |
| `cam:{pos:[x,y,z], look:[x,y,z]}` | カメラのハードカット（毎場スナップ、補間なし） |
| `act:[…]` | 所作の適用（下記3.3） |
| `flash:true, flashPhase:"intro|laugh|hush"` | 歌合舞台の借用場面（能は歌詠み専用の場転換） |

### 3.3 act に書ける所作
- `{who:"key", to:[x,y,z], yaw, emote:"…", t:秒}` — 立ち位置・向き・所作
- **emote 一覧（10種）**: `listen` 聞き入る / `startle` はっとする / `recite` 詠唱 / `laugh` 笑う / `anger` 怒る / `bow` 会釈 / **`weep` 泣く / `nod` 頷く / `point` 指し示す / `shy` 恥じらう**（後4種は波Lで追加）
- 道具制御（buildProps:"chomon" の場合）: `{koushi:"up|half|down"}` 半蔀 / `{meibo:"show|hide"}` 名簿 / `{hitatare:"out|hide"}` 直垂 / `{cricket:"show|hide"}` キリギリス

### 3.4 演出系（波Lで追加。ビートに書くだけで発動）
| フィールド | 内容 | 制約 |
|---|---|---|
| `fade:"black"\|"white"` | 場面転換の暗転/白転（瞬時に覆い0.9秒で明ける） | fxと併用不可 |
| `fx:"lightning"` | 稲光（白の二段明滅＋遠雷音）。REDUCED_MOTION時は音のみ | fadeと併用不可 |
| `se:"gong"\|"bell"\|"koto"\|"thunder"\|"wind"` | 効果音キュー（WebAudio合成・外部音源なし） | — |
| `chapter:"第一章 ○○"` | 章題カード（中央大表示2.4秒） | wakaと併用不可 |
| bubbles の `cls:"think"` | **心内語**（点線・淡紫・斜体） | — |

既存の bubbles cls: `""`(通常) / `laugh`(朱・強調) / `anger`(怒) / `hush`(沈黙・灰) / `name`(名札・紺)。

## 4. 大規模ストーリーの推奨構造

- **章 → 場 → ビート** の三層。1章＝8〜15ビート、章頭ビートに `chapter` ＋ `fade:"black"` を置くと「幕が開く」リズムが出る。
- 長編は `SAIGEN_SCENES` に**章ごとに1シーン**として登録し、`unit:"場"` にすると目次がそのまま章立てになる。終演メニュー「他の話を読む」が次章への導線になる。
- 新しい場所（門前・釣殿・牛車など）を使う時は `world:true` ＋ cast の `pos` に実座標を書く。専用の大道具が要る時だけ `SAIGEN_PROP_BUILDERS` に builder を1つ追加（`chomon` が実装例: 大型御簾・半蔀・名簿・直垂・キリギリス）。
- カメラは**1ビート1カット**が原則（紙芝居）。同一構図の連続は2ビートまで。

## 5. 既存シナリオ校閲記録（2026-07-02 総監督レビュー）

### 修正済み
1. **能は歌詠み 第1段の注**: 「花園の左大臣＝源有仁。邸が花園にあった」を追記（人物の同定は入試頻出）。
2. **第8段の女房の吹き出し**: 「ふふっ、春の歌？」→「まあ……春の歌を」。現代口語の「？」を排し、hush場の抑制された文体と統一。

### 検討の上、変更しなかった点（理由つき）
- **第2段本文「愛しておはしましけるに」**: 教科書によっては「愛で（めで）おはしましけるに」とする伝本もあるが、「愛す（サ変）」も中古に用例があり、底本差の範囲。**出典を確認せず本文をいじるのは危険**なので保留。変更する場合は採用教科書の本文と照合すること。
- 枕草子第一段（4ビート）: 本文・訳・注とも通行本と一致。修正なし。
- 寛平歌合の段（12〜15）: 「同じことにや」までの本文・訳・注は適正。speaker の付け方（引用台詞の話者を入れる）は第3段以来の慣例と一致しており、統一されている。

### 既知の演出上の弱点（次の改善候補）
- 歌合の場（flash）は歌合モードの舞台の借用のため、camを変えても人物レイアウトは固定。専用ストーリーで多用するなら flash 相当の汎用「別舞台」機構が要る。
- 仮設舞台（world:false）は几帳と茵のみで簡素。長編で使うなら buildProps を1つ作って飾ること。

## 6. 検証手順

1. `node scripts/verify-html.js` → status:ok を確認
2. scriptブロックを抽出して `node --check`
3. シーン追加時は最低限、目次→全ビート送り→終演→「他の話を読む」→トップ帰還 を目視（Playwright不通環境では静的確認のみで、実機確認をユーザーに依頼）
